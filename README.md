# Suno AI 音乐文本质量评估与自动优化实施手册

## ——「陈奕迅歌词工程化 × Suno 适配」全流程体系（V2.0 优化版）


### 前言

本手册是一套**从评估到优化的端到端实施体系**，融合了两大核心方法论：

1. **Suno AI 音乐文本质量评估体系**：28 个维度 + 四变量数学模型，覆盖 Styles 提示词技术参数与 Lyrics 歌词艺术质量的双重评估。

2. **自动优化系统架构**：基于评估结果的问题自动诊断、评分预测、针对性修正、迭代校准的完整工作流。

**V2.0 优化重点**：
- 细化四变量数学模型的可计算锚点，降低人工判断模糊性。
- 新增「温度参数」与「生成随机性控制」模块。
- 建立「失败案例库」与反向校准机制。
- 补充自动化工具的优先级与实施路径。

本手册适用于：
- AI 音乐生成前的 Prompt 质量预检与自动优化
- 批量生成作品的自动化筛选与评分
- 不同版本歌词与风格提示词的 A/B 测试
- 与人类顶级作品的对标分析
- 自动化优化工具的开发参考

**核心设计原则**：
1. **量化颗粒度**：每个维度的每个分数档位均有可逐项勾选的量化指标。
2. **可计算的情感**：通过四变量数学模型（PAC/TSMI/NSRQ/C3AC）实现情感能量的量化评估。
3. **自动化导向**：所有评分标准均可转化为程序判断逻辑。
4. **双语支持**：同时支持粤语与普通话歌词的工程化评估与优化。
5. **闭环反馈**：内置生成后音频反馈校准模块，持续优化评分模型。

### 第一章：体系总览

#### 1.1 二十八维度评估框架

| 评估对象 | 维度编号 | 评估内容 | 满分 |
| :--- | :--- | :--- | :--- |
| **Styles 文本** | V1–V14 | 人声标签、风格标签、配器、混音、空间、低频、高频、动态、结构、Hook、情绪、人声前置、排除项、综合 | 100 分（加权） |
| **Lyrics 文本** | V15–V28 | 主题选择、物化意象、叙事细节、动词精准、口语底色、Hook 设计、结构建筑、Bridge 设计、尾奏处理、演唱质感、编曲配置、留白负空间、声调协音、动态弧线 | 470 分（粤）/ 465 分（普） |
| **情感模型** | TEE | 寄生动作链、声调匹配、负空间呼吸、副歌 3 弧线 | 470/465 分 |

#### 1.2 总分计算公式

```
最终总分 = Styles加权分 × 0.4 + Lyrics加权分 × 0.6 + B_offset + E_bonus + TEE_offset
```

**等级划分**：

| 最终总分 | Lyrics 部分得分 | 等级 | 商用建议 |
| :--- | :--- | :--- | :--- |
| ≥95 | ≥450/470（粤）或≥445/465（普） | S+（封神级） | 可直接发行 |
| 85–94 | 400–449（粤）或 395–444（普） | S（优秀级） | 可直接商用 |
| 75–84 | 350–399（粤）或 345–394（普） | A（合格级） | 建议微调后商用 |
| 60–74 | <350（粤）或<345（普） | B（待改进） | 需重点优化 |
| <60 | — | C（不合格） | 不可用 |

### 第三章：Lyrics 部分评分标准（V15–V28）

#### V15 主题选择（30 分）

| 分数档位 | 量化标准 | 扣分规则 |
| :--- | :--- | :--- |
| **25–30 分** | 主题来自具体日常瞬间；具备都市人共同经验；有压抑×释放的情感张力 | 宏大叙事扣 5 分；过于私人化扣 5 分；单一时态扣 5 分 |
| **15–24 分** | 主题有一定日常性但不够聚焦；共鸣面偏窄；情感张力单一 | — |
| **<15 分** | 抽象概念主题；缺乏共鸣基础；无情感张力设计 | 直接判为不合格 |

#### V16 物化意象（35 分）

**核心法则**：把抽象情绪物化成一个具体的东西，这个东西必须可以「戴上/除下/裂开/长进肉里」。

| 分数档位 | 量化标准 | 扣分规则 |
| :--- | :--- | :--- |
| **30–35 分** | 有贯穿全曲的核心物象；物象有完整动作链（出现→接触→转折→落点）；物象承载明确情感重量 | 无核心物象扣 10 分；动作链不完整扣 5 分；物象与情感脱节扣 5 分 |
| **20–29 分** | 有核心物象但不够贯穿；动作链部分缺失；情感关联较弱 | — |
| **<20 分** | 无核心物象或仅为背景装饰；静态描述无动作 | 严重扣分 |

#### V17 叙事细节（35 分）

**核心法则**：不说「我很痛」，只呈现画面和动作。必须有三个以上具体的、反抒情的细节动作。

| 分数档位 | 量化标准 | 扣分规则 |
| :--- | :--- | :--- |
| **30–35 分** | 有 ≥3 个具体动作；至少有 1 个「被动发现」瞬间；有日常物件细节 | 每少一个具体动作扣 5 分；主动反思扣 5 分 |
| **20–29 分** | 有 1–2 个具体动作；被动发现较弱；细节略显空洞 | — |
| **<20 分** | 大量情绪描述词汇；缺乏画面感 | 严重扣分 |

#### V18 动词精准（25 分）

**核心法则**：必须有一个让人心头一紧的动词。

| 分数档位 | 量化标准 | 扣分规则 |
| :--- | :--- | :--- |
| **20–25 分** | 核心动词暴力/精准；动词「只呈现不解释」 | 动词平淡扣 10 分；动词后跟解释性语句扣 5 分 |
| **10–19 分** | 动词准确但不够「狠」；有轻微解释倾向 | — |
| **<10 分** | 动词平淡常规；大量解释性语言 | 严重扣分 |

**动词词库**：
- 粤语：㩒、搣、攥、渗、抰、揿、捽、搁、𢱕、𠺘
- 普通话：撕、攥、渗、按、扯、搁、关、揉、钉、刻、埋、吞

#### V19 口语底色（粤 25 分/普 20 分）

**粤语版**（满分 25）：

| 分数档位 | 量化标准 | 扣分规则 |
| :--- | :--- | :--- |

### 第四章：四变量数学模型（V2.0 优化版）

#### 4.1 寄生动作链系数（PAC）

**定义**：物象动作链中，「寄生节点」的数量与质量所决定的张力值。

```
PAC = Σ (寄生节点权重 × 寄生形态系数)
```

**寄生形态系数**（优化后增加明确文本锚点）：

| 形态 | 系数 | 文本特征锚点（判定规则） |
| :--- | :--- | :--- |
| 触感寄生 | 1.5 | 歌词中出现温度/触觉词（冷/热/刺/软/硬/黏）且主语为物象 |
| 空间寄生 | 1.2 | 物象位置与身体部位形成对应（如「挂在脖子上」「贴在胸口」） |
| 时间寄生 | 1.8 | 物象动作触发时间回溯词（记得/那年/曾经/旧/褪色/磨损） |
| 无寄生 | 0 | 物象仅在自身逻辑内运动，无上述任何锚点 |

**寄生节点权重**：

| 节点位置 | 权重 | 判定条件 |
| :--- | :--- | :--- |
| 物象初始状态 | 0.5 | 物象首次出现的描述 |
| 与人接触节点 | 1.0 | 物象与人产生物理或感官接触 |
| 动作转折点 | 1.5 | 物象状态发生改变（裂开/掉落/变轻/变重） |
| 最终落点 | 2.0 | 物象在尾奏处的最终状态 |

**PAC 阈值与评分映射**：

| PAC 区间 | V16 物化意象维度得分调整 |
| :--- | :--- |
| ≥5.0 | +5 分 |
| 3.0–4.9 | +0～+3 分 |
| 1.0–2.9 | -3～-5 分 |
| <1.0 | -8 分 |

#### 4.2 声调-风格匹配度（TSMI）

```
TSMI = 声调基础分 × 风格匹配系数 × 位置权重
```

**风格匹配系数（粤语版）**：

| 风格 | 最佳声调（系数） | 禁用声调（系数为负） |
| :--- | :--- | :--- |
| 慢歌钢琴叙事 | 阴上 ×1.5 / 阴平 ×1.3 | 全部入声 ×(-1.0) |
| Funky 中板冷感 | 阴入 ×1.5 / 阴去 ×1.3 | 阳平 ×0.5 |
| Synth-pop 舞曲 | 阴去 ×1.5 / 阴入 ×1.3 | 阳平 ×0.6 |
| 民谣叙事 | 阳平 ×1.3 / 阴平 ×1.2 | 入声 ×0.8 |
| Band Sound 撕裂 | 阴去 ×1.5 / 阴平 ×1.2 | 阳上 ×0.5 |
| 歌剧式管弦乐 | 阴平 ×1.5 / 阳平 ×1.3 | 入声 ×0.7 |

**风格匹配系数（普通话版）**：

| 风格 | 最佳声调（系数） | 禁用声调（系数为负） |
| :--- | :--- | :--- |
| 慢歌钢琴叙事 | 第三声 ×1.5 / 第一声 ×1.3 | 第四声 ×0.3 |
| Funky 中板冷感 | 第四声 ×1.5 / 第一声 ×1.2 | 第三声 ×0.5 |
| Synth-pop 舞曲 | 第四声 ×1.5 / 第一声 ×1.3 | 第三声 ×0.4 |
| 民谣叙事 | 第三声 ×1.5 / 第一声 ×1.3 | 第四声 ×0.5 |
| Band Sound 撕裂 | 第四声 ×1.5 / 第一声 ×1.2 | 第三声 ×0.4 |
| 歌剧式管弦乐 | 第一声 ×1.5 / 第二声 ×1.3 | 第四声 ×0.6 |

**位置权重**：

| 位置 | 权重 |
| :--- | :--- |
| Hook 尾字 | ×2.0 |
| 副歌内部句尾 | ×1.2 |
| 主歌句尾 | ×1.0 |

**TSMI 阈值与评分映射**：

| TSMI 区间 | V20 Hook 设计维度得分调整 |
| :--- | :--- |
| ≥15 | +5 分 |
| 8–14 | 0～+3 分 |
| 0–7 | -3～-5 分 |
| <0 | -10～-17 分 |

#### 4.3 负空间呼吸量（NSRQ）

```
NSRQ = Σ(停顿实际时长 / 停顿理想时长) × 停顿位置权重
```

**理想呼吸递进序列**（慢歌/歌剧）：

| 位置 | 理想时长 | 位置权重 | 文本标注要求 |
| :--- | :--- | :--- | :--- |
| 骤停后第一口气 | 0.5 秒 | 0.5 | `[pause 0.5s]` |
| 画面陈述后悬停 | 2.0 秒 | 1.5 | `[pause 2.0s]` |
| 揭穿真相前沉默 | 3.0 秒 | 2.0 | `[pause 3.0s]` |
| 念白后收尾 | 1.0–1.5 秒 | 0.8 | `[pause 1.0s]` 或 `[pause 1.5s]` |
| 尾奏完全沉默 | 5.0 秒 | 1.0 | `[silence 5.0s]` |

**NSRQ 阈值与评分映射**：

| NSRQ 区间 | V22 Bridge 设计维度得分调整 |
| :--- | :--- |
| ≥4.0 | +3 分 |
| 2.5–3.9 | 0～+1 分 |
| 1.0–2.4 | -2～-4 分 |
| <1.0 | -5～-8 分 |

#### 4.4 副歌 3 双阶弧线完成度（C3AC）

```
C3AC = 识穿阶得分 × 认领阶得分
```

**识穿阶**（0 或 1）：
- 副歌 3 首句或次句包含识穿表述 = 1，否则 = 0
- 粤语识穿词库：我知、原来、其实、不过、只系、终于、到头来
- 普通话识穿词库：原来、其实、不过是、我终于知道、后来才懂、到头来

**认领阶**（0 / 0.5 / 1）：
- 副歌 3 末句包含认领动词 + 物象关联 = 1
- 仅有认领动词但无物象关联 = 0.5
- 无认领动词 = 0
- 粤语认领词库：留低、唔抆、等、替、当系、由佢、揸住、揽实、继续、仍然
- 普通话认领词库：由它、留着、继续、就当是、让它、放着、等着、还是、依然

**C3AC 阈值与评分映射**：

| C3AC 区间 | V21 结构建筑维度得分调整 |
| :--- | :--- |
| ≥1.5 | +5～+8 分 |
| 1.0–1.49 | 0～+3 分 |
| 0.5–0.99 | -3～-5 分 |
| <0.5 | -8 分 |

#### 4.5 TEE 总情感能量函数

```
粤语版：TEE = (PAC × 3.0) + (TSMI × 2.0) + (NSRQ × 7) + (C3AC × 22) + 380
普通话版：TEE_普 = (PAC × 3.0) + (TSMI_普 × 2.0) + (NSRQ × 7) + (C3AC × 22) + 375
```

**TEE_offset 计算**：

```
TEE_offset = (TEE / 470 或 465) × 100 - Lyrics加权百分制
```

TEE_offset 作为情感模型的修正值，直接加到最终总分中。

### 第六章：失败案例库与反向校准机制（V2.0 新增）

#### 6.1 高频失败模式识别表

| 失败模式 | 典型表现 | 文本层面原因 | 自动修正策略 |
| :--- | :--- | :--- | :--- |
| **情绪误读** | 悲伤歌词被唱成欢快 | Styles 中情绪标签与歌词冲突（V11 一票否决未触发） | 强制检查 V11 匹配度，冲突时自动添加情绪纠正标签 |
| **Bridge 混响残留** | Bridge 人声仍带明显混响 | 未标注 `NO reverb on bridge vocals` | 自动在 Styles 中添加该排除标签 |
| **副歌 3 旋律走样** | 副歌 3 完全重写旋律，失去记忆点 | C3AC=0，识穿阶/认领阶缺失 | 自动补全副歌 3 的识穿句与认领动词 |
| **粤语发音怪异** | 粤语歌词被用普通话声调演唱 | TSMI<0，Hook 尾字入声却被分配长音 | 自动替换 Hook 尾字为阴上/阴平 |
| **动态失衡** | 全曲音量一致，无起伏 | V8 动态标签缺失，V28 动态弧线未标注 | 自动按标准弧线添加动态标注 |
| **尾奏冗长** | Outro 出现总结性语句或多余乐器 | V23 尾奏处理扣分项触发 | 自动截断尾奏，仅保留一个动作 + `[silence 5.0s]` |
| **人声被乐器淹没** | 副歌部分人声听不清 | V12 人声前置量缺失或数值偏小 | 自动添加 `vocals mixed 2.5dB above instruments` |
| **频率刺耳** | 高频过于尖锐 | V7 高频标签缺失 | 自动添加 `smooth high end`、`no harsh frequencies` |

#### 6.2 反向校准流程

```
生成音频 → 人工/自动评分 → 对比文本预测评分
                ↓
        偏差分析（偏差 > 10 分）
                ↓
    记录到失败案例库 → 触发对应维度权重微调
                ↓
        更新下一轮评分预测模型
```

**校准参数**：

| 偏差类型 | 校准动作 |
| :--- | :--- |
| 文本高分但音频低分 | 降低该风格下相关维度的权重 5%，增加随机性控制标签 |
| 文本低分但音频高分 | 检查是否有未覆盖的质量特征，补充评分项 |
| 某风格持续偏差 | 为该风格单独建立权重偏移表 |

### 第九章：Lyrics 模块自动优化规则

#### 9.1 结构标签规范化

**规则**：
1. 将所有不规范标签转换为标准格式：`[Verse 1]`、`[Chorus]`、`[Bridge]`、`[Outro]` 等
2. 检查结构完整性，缺失段落自动提示
3. 确保每个标签独占一行，标签后紧跟歌词内容

#### 9.2 C3AC 副歌 3 双阶弧线自动补全

**输入**：Lyrics 文本、语言

**输出**：优化后的副歌 3 段落

**步骤**：
1. 提取副歌 3 段落（识别 `[Chorus 3]` 或 `[Final Chorus]`）
2. 检查识穿句：
   - 粤语：搜索 `我知`、`原来`、`其实`、`不过`、`只系`、`终于`、`到头来`
   - 普通话：搜索 `原来`、`其实`、`不过是`、`我终于知道`、`后来才懂`、`到头来`
   - 若无，在副歌 3 首句或次句位置插入识穿句模板（如「原来我早该知道」/「我知这一切不过」）
3. 检查认领动词：
   - 粤语：搜索 `留低`、`唔抆`、`等`、`替`、`当系`、`由佢`、`揸住`、`揽实`、`继续`、`仍然`
   - 普通话：搜索 `由它`、`留着`、`继续`、`就当是`、`让它`、`放着`、`等着`、`还是`、`依然`
   - 若无，在副歌 3 末句插入认领句模板，需关联核心物象

#### 9.3 NSRQ 负空间自动标注

**输入**：Lyrics 文本、风格

**输出**：标注了精确停顿的 Lyrics

**步骤**：
1. 定位 Bridge 段落
2. 在 Bridge 开头插入 `[All Instruments Cut]` 和 `[Naked Voice]`
3. 按呼吸序列插入停顿：
   - Bridge 首句后：`[pause 0.5s]`
   - Bridge 中间：`[pause 2.0s]`
   - Bridge 倒数第二句后：`[pause 3.0s]`
4. 在 Outro 末尾添加 `[silence 5.0s]`

#### 9.4 动态弧线自动标注

**输入**：Lyrics 文本、风格

**输出**：标注了完整动态弧线的 Lyrics

**步骤**：
1. 识别各段落类型
2. 按标准动态弧线添加标注：
   - `[Intro]` 后：`[Very Soft]`
   - `[Pre-Chorus 1]` 前：`[Building]`
   - `[Chorus 1]` 前：`[Released But Held Back]`
   - `[Verse 2]` 前：`[Slightly Heavier]`
   - `[Pre-Chorus 2]` 前：`[Building Stronger]`
   - `[Chorus 2]` 前：`[More Released]`
   - `[Bridge]` 后：`[Naked Voice]`
   - `[Chorus 3]` 前：`[Full Voice, Cracks Open]`
   - `[Outro]` 前：`[Falls to Breath]`

#### 9.5 TSMI 声调匹配度自动检测与建议

**输入**：Hook 尾字、风格、语言

**输出**：匹配度评分及替换建议

**步骤**：
1. 提取 Hook 最后一个汉字
2. 获取其声调及基础分
3. 根据风格获取匹配系数
4. 计算 TSMI 值
5. 若 TSMI < 5，从同韵部中推荐基础分更高的同义字

#### 9.6 动词精准度自动提升

**输入**：Lyrics 文本、语言

**输出**：动词优化建议

**规则**：
1. 识别核心动作词（位于物象动作链上的动词）
2. 与动词词库对比：
   - 粤语词库：㩒、搣、攥、渗、抰、揿、捽、搁、𢱕、𠺘
   - 普通话词库：撕、攥、渗、按、扯、搁、关、揉、钉、刻、埋、吞
3. 若为平淡动词（如「放」「拿」「走」），推荐替换为词库中的暴力精准动词

### 附录 A：关键法则速记

1. **写动作不写情绪**
2. **物象必须有动作链**（出现→接触→转折→落点），且至少一个节点「寄生」到人
3. **Hook ≤ 4 字，重复 ≥ 4 次**，尾字声调与风格匹配
4. **Bridge = 全曲最脆弱处**：乐器骤停、裸声、一个画面、不解释
5. **副歌 3 必须完成「识穿→认领」弧线**
6. **尾奏 = 一个动作 + 沉默**，不给答案
7. **口语底色**：读出来像一个人在说话
8. **动词暴力精准**
9. **Custom Mode 是底线**：关闭 Auto Instrumental 和 Auto Structure
10. **停顿必须带秒数**：`[pause 0.5s]` 而非 `[停顿]`
11. **Temperature 按风格调整**：慢歌低、摇滚高
12. **失败案例库驱动迭代**：每次音频生成后记录偏差，反向校准评分模型
