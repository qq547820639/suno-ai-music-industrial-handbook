# AI 音乐文本质量评估与自动优化执行引擎

## ——「歌词工程化 × Suno 适配」全流程体系（v2.1 正式版）

---

### 前言

本手册是执行引擎 v2.1 的正式发布版本。v2.0 为结构升级版，v2.1 在此基础上完成了数学模型的全面审计与冻结。

**v2.1 核心变更**：

1. **C3AC 三变量弧线公式**：废弃旧二元公式（识穿阶 × 认领阶，最大值 1.0 但阈值要求 ≥1.5，数学不可达），替换为 `C3AC = 0.6D + 0.7C + 0.7L`，值域 [0, 2.0]，新增延续阶 L。
2. **TEE 归一化交互模型**：废弃旧加法模型，替换为含交互项的六项加权模型 + 归一化变量。
3. **Gate 改为结构性扣分函数**：废弃旧 Gate 连乘直接乘 TEE_raw 的"强淘汰器"模式，替换为加权几何均值 G_struct + SoftPenalty + CriticalPenalty 的三层扣分架构。
4. **PAC 归一化分母修正**：从 14.4 改为 9.0（旧分母导致 PAC_n 最大值仅 0.625，永远无法达到 1.0）。
5. **CriticalPenalty 全部统一为 clamp**：避免 TSMI 负值溢出。
6. **ln(0) 实现防呆**：G_struct 计算加入零值保护。
7. **temperature 从 Styles 标签中移除**：改为生成参数推荐，不写入 Suno Styles 文本。
8. **"自动收敛"改为"有限闭环"**：明确单次回复的迭代能力边界。
9. **新增工程防呆模块**：Styles 长度适配、粤语 Unicode 兼容性、创作者偏见缓解机制。
10. **双版本基础常数差异化**：粤语 340 / 普通话 335。

#### 1.2 总分计算公式

```
最终总分 = Styles加权分 × 0.4 + Lyrics加权分 × 0.6 + TEE_offset + B_offset + E_bonus
```

TEE_offset 的计算方式见第四章 v2.1 数学模型。

| 最终总分 | 等级 | 商用建议 |

### 第四章：四变量数学模型（v2.1 冻结版）

> 本章为 v2.1 冻结版本。所有公式、参数、阈值均经审计确认，可直接用于实现。

#### 4.1 寄生动作链系数（PAC）

**定义**：物象动作链中，「寄生节点」的数量与质量所决定的张力值。

```
PAC_raw = Σ (寄生节点权重 × 寄生形态系数)
```

**寄生形态系数**（含明确文本锚点）：

| 形态 | 系数 | 文本特征锚点（判定规则） |
| :--- | :--- | :--- |
| 触感寄生 | 1.5 | 歌词中出现温度/触觉词（冷/热/刺/软/硬/黏）且主语为物象 |
| 空间寄生 | 1.2 | 物象位置与身体部位形成对应（如「挂在脖子上」「贴在胸口」） |
| 时间寄生 | 1.8 | 物象动作触发时间回溯词（记得/那年/曾经/旧/褪色/磨损） |
| 无寄生 | 0 | 物象仅在自身逻辑内运动，无上述任何锚点 |

**寄生节点权重**：

| 节点位置 | 权重 | 判定条件 |
| :--- | :--- | :--- |
| 物象初始状态 | 0.5 | 物象首次出现的描述 |
| 与人接触节点 | 1.0 | 物象与人产生物理或感官接触 |
| 动作转折点 | 1.5 | 物象状态发生改变（裂开/掉落/变轻/变重） |
| 最终落点 | 2.0 | 物象在尾奏处的最终状态 |

**PAC_raw 理论最大值**：4 节点全满 × 时间寄生 = (0.5 + 1.0 + 1.5 + 2.0) × 1.8 = **9.0**

**PAC_raw 阈值与 V16 物化意象维度得分映射**：

| PAC_raw 区间 | V16 得分调整 |
| :--- | :--- |
| ≥5.0 | +5 分 |
| 3.0–4.9 | +0～+3 分 |
| 1.0–2.9 | -3～-5 分 |
| <1.0 | -8 分 |

#### 4.3 负空间呼吸量（NSRQ）

```
NSRQ_raw = Σ(停顿实际时长 / 停顿理想时长) × 停顿位置权重
```

**慢歌/歌剧理想呼吸递进序列**：

| 位置 | 理想时长 | 位置权重 | 文本标注要求 |
| :--- | :--- | :--- | :--- |
| 骤停后第一口气 | 0.5 秒 | 0.5 | `[pause 0.5s]` |
| 画面陈述后悬停 | 2.0 秒 | 1.5 | `[pause 2.0s]` |
| 揭穿真相前沉默 | 3.0 秒 | 2.0 | `[pause 3.0s]` |
| 念白后收尾 | 1.0–1.5 秒 | 0.8 | `[pause 1.0s]` 或 `[pause 1.5s]` |
| 尾奏完全沉默 | 5.0 秒 | 1.0 | `[silence 5.0s]` |

**快歌（Funky/Synth-pop/Rock）理想呼吸序列**：

| 位置 | 理想时长 | 位置权重 | 文本标注要求 | 说明 |
| :--- | :--- | :--- | :--- | :--- |
| Breakdown 后第一拍 | 0.3 秒 | 0.5 | `[pause 0.3s]` | 紧凑呼吸 |
| 副歌前蓄力 | 0.8 秒 | 1.0 | `[pause 0.8s]` | 不拖沓 |
| Bridge 破裂前 | 1.5 秒 | 2.0 | `[pause 1.5s]` | 快歌中的"长沉默" |
| 最后一个音后 | 1.0 秒 | 0.8 | `[pause 1.0s]` | 快歌尾奏不需要 5 秒沉默 |
| 骤停 | 0 秒 | 1.0 | `[Abrupt Stop]` | 快歌尾奏 = 骤停 |

**NSRQ_raw 理论最大值（慢歌）**：(0.5/0.5)×0.5 + (2.0/2.0)×1.5 + (3.0/3.0)×2.0 + (1.5/1.5)×0.8 + (5.0/5.0)×1.0 = **5.8**

**NSRQ_raw 理论最大值（快歌）**：(0.3/0.3)×0.5 + (0.8/0.8)×1.0 + (1.5/1.5)×2.0 + (1.0/1.0)×0.8 + (0/0)×1.0 = **5.3**

**NSRQ_raw 阈值与 V22 Bridge 设计维度得分映射**：

| NSRQ_raw 区间 | V22 得分调整 |
| :--- | :--- |
| ≥4.0 | +3 分 |
| 2.5–3.9 | 0～+1 分 |
| 1.0–2.4 | -2～-4 分 |
| <1.0 | -5～-8 分 |

#### 4.4 副歌弧线完成度（C3AC）——三变量公式（v2.1）

```
C3AC = 0.6D + 0.7C + 0.7L
```

| 变量 | 名称 | 值域 | 定义 |
| :--- | :--- | :--- | :--- |
| D | 识穿阶 | {0, 1} | 副歌 3 首句或次句包含识穿表述 = 1，否则 = 0 |
| C | 认领阶 | {0, 0.5, 1} | 末句包含认领动词 + 物象关联 = 1；仅有认领动词无物象 = 0.5；无认领动词 = 0 |
| L | 延续阶 | {0, 0.5, 1} | 见下方定义 |

**C3AC 理论值域**：[0, 2.0]（D=1, C=1, L=1 时取最大值 2.0）

**识穿词库**：
- 粤语：我知、原来、其实、不过、只系、终于、到头来
- 普通话：原来、其实、不过是、我终于知道、后来才懂、到头来

**认领词库**：
- 粤语（默认弧线）：留低、唔抆、等、替、当系、由佢、揸住、揽实、继续、仍然
- 普通话（默认弧线）：由它、留着、继续、就当是、让它、放着、等着、还是、依然
- 否认型弧线：放开、算了、走吧、不再、忘了、丢掉
- 觉醒型弧线：不再需要、已经够了、自由、重生

**延续阶 L 定义**：

| 值 | 文本判断标准 |
| :--- | :--- |
| **L = 1** | 副歌 3 保留副歌 1/2 的核心句式、Hook 位置、重复节奏或关键词骨架，且最后两句完成从"识穿"到"认领"（或其他弧线模板的首尾阶段）的自然过渡 |
| **L = 0.5** | 保留了部分句式/Hook/关键词骨架，但语义过渡不完整；或语义过渡完整，但副歌 3 与前两次副歌的文本骨架明显偏离 |
| **L = 0** | 副歌 3 完全重写，Hook 消失，或语义与识穿/认领（或其他弧线模板）无关 |

**弧线模板选择**（根据歌曲情感走向匹配）：

| 弧线模板 | 首阶段关键词 | 末阶段关键词 | 适用情感 |
| :--- | :--- | :--- | :--- |
| 识穿→认领（默认） | 我知/原来/其实 | 留低/由佢/继续 | 接受、认命、包容 |
| 否认→放手 | 不是/没有/才不是 | 放开/算了/走吧/忘了 | 决绝、释然 |
| 困惑→坚持 | 不懂/为什么/怎么会 | 还是/仍然/继续/一直 | 执念、迷恋 |
| 觉醒→超越 | 终于明白/我看见 | 不再需要/已经够了/自由 | 顿悟、解脱 |

**C3AC_raw 阈值与 V21 结构建筑维度得分映射**：

| C3AC_raw 区间 | V21 得分调整 |
| :--- | :--- |
| ≥1.5 | +5～+8 分 |
| 1.0–1.49 | +0～+3 分 |
| 0.5–0.99 | -3～-5 分 |
| <0.5 | -8 分 |

#### 4.5 TEE 总情感能量函数（v2.1 冻结版）

##### Step 1：归一化

```
PAC_n  = clamp(PAC_raw  /  9.0,  0,  1)
TSMI_n = clamp(TSMI_raw / 30.0, -1,  1)
NSRQ_n = clamp(NSRQ_raw /  5.0,  0,  1)
C3AC_n = clamp(C3AC_raw /  2.0,  0,  1)
```

其中 `clamp(x, lo, hi) = min(max(x, lo), hi)`

##### Step 2：TEE_raw 计算

```
TEE_raw = base
        + 32·PAC_n
        + 24·TSMI_n
        + 20·NSRQ_n
        + 28·C3AC_n
        + 16·(PAC_n × TSMI_n)
        + 12·(NSRQ_n × C3AC_n)
```

| 版本 | base | 歌词满分（LyricsFullScore） |
| :--- | :--- | :--- |
| 粤语 | 340 | 470 |
| 普通话 | 335 | 465 |

```
TEE_pct_raw = clamp((TEE_raw / LyricsFullScore) × 100, 0, 100)
```

**TEE_raw 各项贡献范围**：

| 项 | 类型 | 最大贡献 | 设计意图 |
| :--- | :--- | :--- | :--- |
| base | 常数 | 340 / 335 | 基线情感能量 |
| 32·PAC_n | 个体 | 32 | 物象寄生强度 |
| 24·TSMI_n | 个体（可负） | 24（或 -24） | 声调匹配质量 |
| 20·NSRQ_n | 个体 | 20 | 负空间呼吸量 |
| 28·C3AC_n | 个体 | 28 | 副歌弧线完成度 |
| 16·(PAC×TSMI) | 交互 | 16（或 -16） | 物象×声调协同 |
| 12·(NSRQ×C3AC) | 交互 | 12 | 负空间×弧线协同 |

##### Step 3：Gate 函数（结构性扣分）

```
gPAC  = clamp(0.85 × PAC_n  + 0.15, 0, 1)
gTSMI = clamp(0.85 × TSMI_n + 0.15, 0, 1)
gNSRQ = clamp(0.85 × NSRQ_n + 0.15, 0, 1)
gC3AC = clamp(0.85 × C3AC_n + 0.15, 0, 1)
```

Gate 函数设计：当变量为 0 时输出 0.15（底线保护），当变量为 1 时输出 1.0（满分通过），中间线性增长。

```
若 min(gPAC, gTSMI, gNSRQ, gC3AC) = 0：
  G_struct = 0

否则：
  G_struct = exp(
    0.30·ln(gPAC)
  + 0.25·ln(gTSMI)
  + 0.20·ln(gNSRQ)
  + 0.25·ln(gC3AC)
  )
```

**G_struct 为加权几何均值**，权重 [0.30, 0.25, 0.20, 0.25] 对应 [PAC, TSMI, NSRQ, C3AC]。核心性质：当四项相等时 G_struct = 该共同值；单项低分通过乘法拖垮整体。

**实现防呆**：代码实现时使用 `eps = 1e-8` 保护 ln(0)：
```python
G_struct = exp(
    0.30 * log(max(gPAC, eps))
  + 0.25 * log(max(gTSMI, eps))
  + 0.20 * log(max(gNSRQ, eps))
  + 0.25 * log(max(gC3AC, eps))
)
```

##### Step 4：扣分计算

```
SoftPenalty = 35 × (1 - G_struct)
```

SoftPenalty 范围 [0, 29.75]（G_struct 从 1.0 到 0.15^1.0 ≈ 0.15 时）。

```
CriticalPenalty =
  20 × clamp(-TSMI_n / 0.15, 0, 1)
+ 10 × clamp((0.20 - PAC_n) / 0.20, 0, 1)
+ 10 × clamp((0.25 - C3AC_n) / 0.25, 0, 1)
```

| 项 | 触发条件 | 最大扣分 | 设计意图 |
| :--- | :--- | :--- | :--- |
| TSMI 项 | TSMI_n < -0.15（声调严重错位） | 20 分 | 声调错位是致命风险 |
| PAC 项 | PAC_n < 0.20（物象几乎无寄生） | 10 分 | 无物象寄生则情感无根 |
| C3AC 项 | C3AC_n < 0.25（弧线几乎未完成） | 10 分 | 无弧线则副歌 3 失去意义 |

CriticalPenalty 上限 = 20 + 10 + 10 = **40**。

##### Step 5：最终输出

```
TEE_pct_final = clamp(
  TEE_pct_raw - SoftPenalty - CriticalPenalty,
  0, 100
)

TEE_offset = TEE_pct_final - Lyrics加权百分制
```

##### Step 6：最终总分

```
最终总分 =
  Styles加权分 × 0.4
+ Lyrics加权分 × 0.6
+ TEE_offset
+ B_offset
+ E_bonus
```

| :--- | :--- | :--- | :--- | :--- | :--- | :--- | :--- | :--- | :--- |

### 第六章：失败案例库与反向校准机制

#### 6.1 高频失败模式识别表

| 失败模式 | 典型表现 | 文本层面原因 | 自动修正策略 |
| :--- | :--- | :--- | :--- |
| **情绪误读** | 悲伤歌词被唱成欢快 | Styles 中情绪标签与歌词冲突 | 强制检查 V11 匹配度，冲突时自动添加情绪纠正标签 |

| **副歌 3 旋律走样** | 副歌 3 完全重写旋律 | C3AC L=0，延续阶缺失 | 自动检测 L 阶，低于 0.5 时触发骨架保留建议 |
| **粤语发音怪异** | 粤语歌词被用普通话声调演唱 | TSMI_raw<0，Hook 尾字入声却被分配长音 | 自动替换 Hook 尾字为阴上/阴平 |
| **动态失衡** | 全曲音量一致无起伏 | V8 动态标签缺失，V28 动态弧线未标注 | 自动按标准弧线添加动态标注 |
| **尾奏冗长** | Outro 出现总结性语句 | V23 尾奏处理扣分项触发 | 自动截断尾奏，仅保留一个动作 + `[silence 5.0s]` |
| **人声被淹没** | 副歌部分人声听不清 | V12 人声前置量缺失 | 自动添加 `vocals mixed 2.5dB above instruments` |
| **频率刺耳** | 高频过于尖锐 | V7 高频标签缺失 | 自动添加 `smooth high end`、`no harsh frequencies` |
| **伪动作** | 歌词看似有动作但实为静态描述 | PAC 节点仅有"初始状态"，无接触/转折/落点 | 自动识别"看着/想起/觉得"等伪动作词，替换为物理动作 |

## 第二篇：执行引擎

### 第七章：单次回复执行架构（有限闭环）

#### 7.1 设计原则

本引擎的目标是**在一次 AI 回复中完成从空白到成品的有限闭环**。"有限闭环"意味着：

1. 不承诺自动收敛到全局最优解
2. 在 2–3 轮迭代内逼近局部最优
3. 明确标注未收敛的风险
4. 提供后续迭代方向

#### 7.2 总体架构

```
┌─────────────────────────────────────────────────────────┐
│                   单次回复（Single Reply）                 │
│                   有限闭环（Finite Loop）                   │
│                                                         │

│  ├─ 加载知识库核心规则                                     │
│  ├─ 加载目标风格模板                                      │
│  └─ 加载相关失败案例                                      │
│                                                         │
│  Phase 1: 创作（~2,000 tokens 输出）                       │
│  ├─ 生成完整 Styles 文本                                  │
│  └─ 生成完整 Lyrics 文本（含工程标注）                       │

│  ├─ 28 维度 Full Card 评分                                │
```

#### 7.3 收敛判定引擎

**五条件收敛判定**（满足任一条件即停止迭代，进入 Phase 4）：

| 条件 | 描述 | 阈值 |
| :--- | :--- | :--- |
| C1 | 分数稳定 | \|本轮总分 - 上轮总分\| < 3 分 |
| C2 | 维度稳定 | 无任何维度得分变化 > 2 分 |
| C3 | 目标达成 | 已达到目标等级（默认 S 级，≥85 分） |
| C4 | 轮次上限 | 已完成 3 轮迭代 |
| C5 | 振荡检测 | 连续 2 轮修改同一维度但该维度得分未提升 |

**振荡检测详细逻辑**：

```
对于每个被修改的维度 D：
  if D_round_N.score == D_round_N-1.score
     AND D_round_N-1.score == D_round_N-2.score:
    → 标记 D 为"振荡维度"
    → 将 D 从修正队列中移除
    → 在 Phase 4 的风险预警中注明"D 维度已振荡锁定，需人工介入"
```

**单次回复中的收敛行为**：

```
Phase 3 结束时：
  计算 delta = |Phase3总分 - Phase1总分|
  检查五条件

  if 任一条件满足:
    → 进入 Phase 4，标注"已收敛"
  else:
    → 进入 Phase 4，标注"未完全收敛，建议进一步迭代"
    → 附带下一轮修正方向建议
```

#### 9.2 创作流程

```
Step 1: 物象确定
├─ 从种子中提取核心物象
├─ 验证物象可操作性（能否被"戴上/除下/裂开/长进肉里"）
└─ PAC 预规划：设计 4 节点动作链，确保 ≥2 个有寄生

Step 2: Hook 设计（TSMI 预规划）
├─ Hook ≤ 4 字，尾字声调查表选择
├─ 重复策略：≥4 次，每次情绪递进
└─ 计算预估 TSMI_raw

Step 3: 弧线设计（C3AC 预规划）
├─ 选择弧线模板（识穿→认领 / 否认→放手 / 困惑→坚持 / 觉醒→超越）
├─ 设计副歌 3 的识穿句 + 认领句
├─ 确保 L=1：副歌 3 保留副歌 1/2 的句式骨架
└─ 计算预估 C3AC

Step 4: 负空间设计（NSRQ 预规划）
├─ Bridge 结构设计（骤停位置）
├─ 按风格选择慢歌/快歌 pause 序列
├─ 尾奏设计（一个动作 + silence）
└─ 计算预估 NSRQ_raw

Step 5: 完整文本生成
├─ 生成 Styles 文本（按 V1–V13 标准逐项覆盖）
│   ⚠️ 不写入 temperature 值
│   ⚠️ 不写入具体艺人名
│   ⚠️ 检查字符数（见 9.3 Styles 长度适配）
├─ 生成 Lyrics 文本（按 V15–V28 标准逐项覆盖）
│   ⚠️ 粤语歌词使用粤语 Unicode 字符（见 9.4）
├─ 插入工程标注（动态、停顿、结构标签）
└─ 粤语歌词 Unicode 兼容性检查（见 9.4）
```

#### 9.3 Styles 长度适配

Suno 对 Styles 文本的长度限制因输入方式不同：

| 输入方式 | 字符限制 | 适配策略 |
| :--- | :--- | :--- |
| 网页端 Custom Mode | ~200 字符（可视输入框） | 极限压缩：仅保留核心标签+排除标签 |
| API 调用 | ~1000 字符 | 标准压缩：删除重复语义标签，合并同类 |
| 第三方前端 | 视实现而定 | 按 API 标准处理 |

**压缩规则**：
1. 删除重复语义标签
2. 合并同类标签为短语链（逗号分隔）
3. 保留所有排除标签（NO 开头）
4. 保留 BPM 标注
5. **不保留 temperature**（写入生成参数而非 Styles 文本）

**网页端极限压缩示例**：
```
Cantonese baritone piano ballad 68 BPM, breathy intimate, felt piano only, NO drums synths doubling, bridge bare voice, final chorus voice cracks
```

#### 9.4 粤语 Unicode 兼容性

**问题**：部分粤语专用字（如 㩒、搣、𢱕、𠺘）属于 Unicode 扩展区（CJK Unified Ideographs Extension B/A），某些 TTS 引擎或文本渲染环境可能显示为方块或缺失。

**防呆规则**：

| 情况 | 处理 |
| :--- | :--- |
| 使用扩展 B 区字符（如 𢱕 U+20C55） | 在歌词旁标注等效常用字或解释，确保 AI 模型理解语义 |
| 使用扩展 A 区字符（如 㩒 U+3A52） | 同上 |
| 使用常用粤语口语词（如 冇、嘅、咁、嗰） | 无问题，直接使用 |
| 不确定字符是否在目标平台支持 | 优先使用常用粤语字替代 |

**替代词表**：

| 扩展区字 | Unicode | 替代方案 |
| :--- | :--- | :--- |
| 㩒 | U+3A52 | 按（注：粤语"按"义） |
| 𢱕 | U+20C55 | 捽（U+637D，常用） |
| 𠺘 | U+20E98 | 含住（用短语替代） |

#### 10.2 v2.1 四变量计算流程

**PAC 计算**：

```
输入：歌词全文
Step 1: 提取核心物象名词
Step 2: 定位物象在歌词中的所有出现位置
Step 3: 判断节点类型（初始/接触/转折/落点）→ 分配权重
Step 4: 判断寄生形态（触感1.5/空间1.2/时间1.8/无0）
Step 5: PAC_raw = Σ(权重 × 系数)
Step 6: PAC_n = clamp(PAC_raw / 9.0, 0, 1)
```

**TSMI 计算**：

```
输入：Hook 文本 + 风格 + 语言
Step 1: 提取 Hook 所有版本的尾字
Step 2: 查声调基础分表
Step 3: 查风格匹配系数表
Step 4: TSMI_raw = 声调分 × 系数 × 位置权重
Step 5: TSMI_n = clamp(TSMI_raw / 30.0, -1, 1)
```

**NSRQ 计算**：

```
输入：歌词全文 + 风格
Step 1: 判断风格 → 选择慢歌/快歌理想序列
Step 2: 提取所有 [pause X.Xs] / [silence X.Xs]
Step 3: 匹配理想序列位置
Step 4: NSRQ_raw = Σ(实际/理想 × 位置权重)
Step 5: NSRQ_n = clamp(NSRQ_raw / 5.0, 0, 1)
```

**C3AC 计算**：

```
输入：副歌 3 文本 + 情感走向 + 语言
Step 1: 检查首句/次句识穿词 → D = 0 或 1
Step 2: 检查末句认领动词 + 物象关联 → C = 0/0.5/1
Step 3: 检查副歌 3 与副歌 1/2 的骨架延续 → L = 0/0.5/1
Step 4: C3AC = 0.6D + 0.7C + 0.7L
Step 5: C3AC_n = clamp(C3AC / 2.0, 0, 1)
```

**TEE 计算**：

```
Step 1: TEE_raw = base + 32·PAC_n + 24·TSMI_n + 20·NSRQ_n + 28·C3AC_n
        + 16·(PAC_n × TSMI_n) + 12·(NSRQ_n × C3AC_n)
Step 2: TEE_pct_raw = clamp((TEE_raw / LyricsFullScore) × 100, 0, 100)
Step 3: 计算 gPAC, gTSMI, gNSRQ, gC3AC
Step 4: 计算 G_struct（含 ln(0) 保护）
Step 5: SoftPenalty = 35 × (1 - G_struct)
Step 6: CriticalPenalty（三项 clamp）
Step 7: TEE_pct_final = clamp(TEE_pct_raw - SoftPenalty - CriticalPenalty, 0, 100)
Step 8: TEE_offset = TEE_pct_final - Lyrics加权百分制
```

### 第十一章：Phase 3 — 靶向修正模块（Targeted Fix）

#### 11.1 修正优先级排序

```
第 1 优先：Core 层中触发一票否决的维度（直接 1 分）
第 2 优先：Core 层中得分 < 阈值的维度
第 3 优先：v2.1 CriticalPenalty 触发项（TSMI_n<-0.15, PAC_n<0.20, C3AC_n<0.25）
第 4 优先：Important 层中得分 < 阈值且 delta > 5 的维度
第 5 优先：Important 层中得分 < 阈值（delta ≤ 5）
```

#### 11.2 v2.1 驱动的修正规则

**PAC 驱动修正**：

```
诊断：PAC_n < 0.20（触发 CriticalPenalty 10 分）
修正：
1. 检查动作链节点数 → 补充缺失节点
   ├─ 缺"转折" → 副歌 2 添加物象状态变化（裂/碎/褪/掉/长进肉里）
   └─ 缺"落点" → Outro 添加物象最终状态
2. 检查寄生形态 → 添加触感/空间/时间寄生词
3. 预估修正后 PAC_raw → PAC_n → CriticalPenalty 变化
```

**TSMI 驱动修正**：

```
诊断：TSMI_n < -0.15（触发 CriticalPenalty 20 分）
修正：
1. 提取 Hook 尾字声调
2. 查风格匹配系数表找最优声调
3. 在同韵部中搜索最优声调替换字
4. 替换 → 重新计算 TSMI_raw → TSMI_n
```

**C3AC 驱动修正**：

```
诊断：C3AC < 0.5 或 C3AC_n < 0.25（触发 CriticalPenalty 10 分）
修正：
1. D=0 → 在副歌 3 首句/次句插入识穿词
2. C<1 → 在副歌 3 末句插入认领动词+物象关联
3. L<1 → 确保副歌 3 保留副歌 1/2 的核心句式骨架
   ├─ L=0（完全重写）→ 恢复 Hook 位置和关键词骨架
   └─ L=0.5（部分保留）→ 补全语义过渡
4. 预估修正后 C3AC → C3AC_n → CriticalPenalty 变化
```

**NSRQ 驱动修正**：

```
诊断：NSRQ_raw < 1.0
修正：
1. Bridge 添加 [All Instruments Cut] + [Naked Voice]
2. 按理想序列插入 [pause X.Xs]
3. 检查 Bridge 内容 → 删除解释性语句
4. Outro 添加 [silence 5.0s]
```

#### 11.3 v2.1 变量重算规则

```
修正某维度后，重算受影响变量：

修正 V16（物化意象） → 重算 PAC_raw → PAC_n
修正 V20（Hook设计） → 重算 TSMI_raw → TSMI_n
修正 V22（Bridge设计） → 重算 NSRQ_raw → NSRQ_n
修正 V21（结构建筑）/副歌3文本 → 重算 C3AC → C3AC_n
修正 V27（声调协音）+ Hook尾字被替换 → 重算 TSMI_raw → TSMI_n

任一变量变化后：
→ 重算 TEE_raw
→ 重算 TEE_pct_raw
→ 重算 gPAC/gTSMI/gNSRQ/gC3AC → G_struct → SoftPenalty
→ 重算 CriticalPenalty
→ 重算 TEE_pct_final
→ 重算 TEE_offset
→ 重算 最终总分
```

#### 11.4 收敛判定执行

```
Phase 3 结束时：

delta = |Phase3总分 - Phase1总分|

C1: delta < 3?               → YES/NO
C2: 所有维度变化 ≤ 2?        → YES/NO
C3: 总分 ≥ 目标等级下限?      → YES/NO
C4: 已是第 3 轮?             → YES/NO
C5: 是否有振荡维度?          → YES/NO

判定：
  if 任一 YES → 收敛 → Phase 4 标注"已收敛"
  if 全部 NO  → 未收敛 → Phase 4 标注"未完全收敛"
```

| 第三方 | 视实现 | 字符计数 | 按 API 标准 |

**自动压缩优先级**（从低到高删除）：
1. 删除同义重复标签
2. 合并逗号分隔的同类标签
3. 删除高阶加分标签（如有）
4. 保留排除标签（NO 开头，最后删除）
5. 保留 BPM

#### 15.2 粤语 Unicode 兼容性

**检测流程**：
```
扫描歌词全文
├─ 检测 Unicode 码位
├─ 标记 U+20000 以上字符（扩展 B 区）
├─ 标记 U+3400–U+4DBF 字符（扩展 A 区）
├─ 对每个标记字符查替代词表
└─ 若无替代词，在歌词旁添加注音/注释
```

#### 15.3 创作者偏见缓解

**问题**：AI 模型在生成歌词时可能倾向于训练数据中的高频模式（如用"月光"代表孤独、用"雨"代表悲伤），导致意象同质化。

**缓解机制**：

```
Step 1: 意象去重检查
├─ 扫描已生成歌词中的核心意象
├─ 与"高频 AI 意象清单"比对
│   高频意象清单：月光、雨、风、泪水、黑夜、星辰、玫瑰、翅膀、火焰
├─ 若命中 ≥2 个 → 标记为"意象同质化风险"
└─ 推荐替换为更具体的日常物象

Step 2: 动词去俗检查
├─ 扫描核心动词
├─ 与"AI 高频动词清单"比对
│   高频动词清单：想、看、听、说、觉得、希望、等待、相信、记得、忘记
├─ 若命中 → 推荐替换为词库中的精准动词
```

#### 15.4 振荡检测（补充细节）

```
数据结构：
  oscillation_log = {
    维度名: [Round1分, Round2分, Round3分]
  }

检测规则：
  for each 维度 in oscillation_log:
    if len(scores) >= 3:
      if scores[-1] == scores[-2] == scores[-3]:
        → 标记为"振荡锁定"
        → 从修正队列移除
        → Phase 4 风险预警中标注

特殊情况：
  if scores[-1] != scores[-2] but delta < 1:
    → 标记为"收敛中"（非振荡）
```

### 第十六章：Suno 生成常见错误避坑指南

```
1. 不要在 Styles 中写情绪描述
   ❌ sad, emotional, heartbroken
   ✅ restrained, detached, voice cracks

2. 不要使用具体艺人标签
   ❌ like [某歌手]
   ✅ [语言] male warm baritone, slightly breathy

3. 不要在 Styles 中写 temperature
   ❌ temperature=0.7
   ✅ 将 temperature 作为独立生成参数设置

4. 不要在歌词中使用括号注释
   ❌ （沉默）、（哽咽）、（轻声）
   ✅ [Very Soft]、[pause 2.0s]、[voice cracks]

5. 副歌 3 不要完全重写句式骨架
   ❌ 副歌 3 与副歌 1/2 无任何文本重叠
   ✅ 保留 Hook 位置和关键词骨架，仅改动最后两句

6. Bridge 骤停后的人声必须完全干燥
   ❌ Bridge 人声仍带混响
   ✅ 标注 `NO reverb on bridge vocals`

7. Custom Mode 是必须的
   ✅ Custom Mode + 关闭 Auto Instrumental 和 Auto Structure

8. 粤语歌词必须匹配粤语声调
   ✅ 快歌多用入声/阴去，慢歌多用阴上/阴平

9. 尾奏一刀切断
   ❌ Outro 写总结句
   ✅ 一个动作 + [silence 5.0s]

10. 物象必须贯穿
    ❌ 每段换一个新意象
    ✅ 同一物象从主歌 1 到副歌 3 完成动作链

11. v2.1：CriticalPenalty 优先级高于 SoftPenalty
    ❌ 先优化 NSRQ（仅影响 SoftPenalty 的一个因子）
    ✅ 先修 TSMI（CriticalPenalty 最高 20 分）

12. 停顿必须带秒数
    ❌ [pause] 或 [停顿]
    ✅ [pause 0.5s] 或 [pause 2.0s]

13. 检查 Styles 长度
    ❌ 网页端直接粘贴 1000 字符 Styles
    ✅ 按平台限制压缩 Styles 文本
```

### 附录 A：关键法则速记

1. **写动作不写情绪**
2. **物象必须有动作链**，且至少一个节点「寄生」到人

6. **尾奏 = 一个动作 + 沉默**，不给答案
7. **口语底色**：读出来像一个人在说话
8. **动词暴力精准**
9. **Custom Mode 是底线**
10. **停顿必须带秒数**：`[pause 0.5s]` 而非 `[停顿]`
